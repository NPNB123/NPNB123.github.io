RAG总结

 Posted on 01/18/2025  |  477 words  |  NPNB

RAG（Retrieval-Augmented Generation，检索增强生成）是一种融合检索与生成的技术方法，通过将传统检索式问答系统与自然语言生成技术相结合，显著提升了AI系统在回答自然语言问题时的准确性与可靠性.

​ RAG技术通过结合检索系统和生成模型，有效缓解了大模型中常见的Hallucination（幻觉）问题，即模型生成不准确或无依据的内容。

RAG的基本流程和原理

RAG与传统检索式问答系统不同，它并不依赖于维护一个FAQ库，而是使用了一个存放大量原始文档的知识库。RAG的工作流程主要包括以下几个步骤：

  1. 分割：将文档分割为多个较小的文本单元（chunks）。
  2. 文本向量化：将分割后的文本转换为向量表示。
  3. 向量存储：将向量存储在向量数据库中。
  4. 检索：根据用户输入检索最相关的文本片段。
  5. 生成答案：将检索到的文本片段与用户问题结合，输入生成模型生成最终答案。

###1. 分割

将一篇文档分为多个chunks

分割的目的：

  • 提高检索效率：将文档分割成较小的单元后，检索系统可以更精准地定位与查询相关的部分，而不是返回整篇文档.
  • 增强生成模型的输入质量：生成模型在生成答案时，可以基于更小、更相关的文本单元，从而提高生成内容的准确性和相关性.
  • 降低计算复杂度：处理较小的文本单元比处理整篇文档更高效，尤其是在处理大规模文档集合时

分割方法:

分割方法:

  • 根据句子划分：适用于句子结构清晰的文档。
  • 按照字符数划分：适用于需要严格控制分割长度的场景。
  • 根据字符数划分，结合overlapping window：在按字符数分割的基础上，增加重叠窗口，确保分割后的文本单元之间有部分重叠，避免信息丢失。
  • 递归方法分割：使用递归方法分割文本，确保分割后的文本单元尽可能接近设定的长度，同时保持词语的完整性。
  • 根据语义分割：根据语义分割文本，适用于需要保持语义完整性的场景。

分割出的chunks的长短：

  • 太长：会导致检索精度下降，因为检索系统可能无法准确定位相关信息。
  • 太短：会导致语义不完整，不利于大模型理解上下文。

一些常用的分割方法:

英文的分割

  1. 按句子分割:
import nltk
nltk.download('punkt')
from nltk.tokenizer import sent_tokenize

text = " ... "

chunks = sent_tokenize(text)
  1. 按固定字符数分割
def fixed_length_chunks(text, chunk_size):
    return [text[i:i+chunk_size]for i in range(0, len(text), chunk_size)]

text = " ... "
chunks = fixed_length_chunks(text, 100)
  1. 按固定字符数+overlapping window
def sliding_window_chunks(text, chunk_size, stride):
    return [text[i:i+chunk_size] for i in range(0, len(text), stride)]

chunks = sliding_window_chunks(text, 100, 50) # 100个字符的串, 步长50
  1. 用langchain的递归方法RecursiveCharacterTextSplitter(会尽力逼近设定的长度, 但会使词完整)
from langchain.text_splitter import RecursiveCharacterTextSplitter

text = " ... "
splitter = RecursiveCharacterTextSplitter(
	chunk_size = 100,
    chunk_overlap = 50,
    length_function = len
)

chunks = splitter.split_text(text)

中文的分割

  1. 按句子分割
import re
text = "..."  # 用正则表达式匹配标点符号
# 使用正则表达式分割句子，保留标点符号
sentences = re.split(r'(?<=[。！？])', text)
# 重新组合句子和结尾的标点符号
chunks = [sentence + (punctuation if punctuation else '') for sentence, punctuation in zip(sentences, sentences[1:] + [''])]

2. 文本向量化

​ 文本向量化的目标是将自然语言文本转换为数值向量，便于存储和计算相似度。

​ 文本的向量化通常依赖于Embedding模型, 比如使用OpenAI的Embedding

from openai import OpenAI
client = OpenAI()

def get_embedding(text, model="text-embedding-ada-002"):
    return client.embeddings.create(input = [text], model=model).data[0].embedding

3.向量数据库

使用向量数据库（如FAISS、Pinecone、Weaviate、Milvus）存储向量

这里以Langchain和FAISS的代码为例使用向量数据库, 同时使用WebBaseLoader获取网页信息, 分割向量化后交给FAISS

from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings
from langchain_community.document_loaders import WebBaseLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
loader = WebBaseLoader(
    web_paths=("https://lilianweng.github.io/posts/2023-06-23-agent/",),
    bs_kwargs=dict(
        parse_only=bs4.SoupStrainer(
            class_=("post-content", "post-title", "post-header")
        )
    ),
)

docs = loader.load()
text_splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
all_splits = text_splitter.split_documents(docs)
embeddings = OpenAIEmbeddings(model="text-embedding-ada-002")
vector_store = FAISS(embedding_function=embeddings)
vector_store.add_documents(documents=all_splits)

使用上面得到的向量数据库构建

最后一部即将检索到的片段与用户问题拼接, 作为Prompt输入进大模型

衡量RAG性能的指标

上文说过, RAG技术通过结合检索系统和生成模型，有效缓解了大模型中常见的Hallucination（幻觉）问题虽然RAG明显缓解了幻觉问题, 但有时RAG也会产生幻觉问题. 为了改进RAG的性能, 首先要明确衡量RAG性能的指标

1. 准确率（Accuracy）

定义：准确率衡量检索系统中正确预测的文档（包括相关文档和不相关文档）占总文档的比例。 $$ \text{Accuracy} = \frac{\text{检索到的相关文档数} + \text{未检索到的不相关文档数}}{\text{知识库中总文档数}} $$

适用场景：适用于初步评估检索系统的整体表现，但在大型知识库中可能因不相关文档过多而失真。

2. 精确率（Precision）

定义：精确率衡量检索到的文档中与查询相关的比例，反映检索结果的质量。

$$ \text{Precision} = \frac{\text{检索到的相关文档数}}{\text{检索到的总文档数}} $$

变体：Precision@k
仅考虑前 ( k ) 个检索结果中的相关文档比例： $$ \text{Precision@k} = \frac{\text{前k个结果中相关文档的数量}}{k} $$

适用场景：适用于评估检索结果的质量，尤其是当仅使用前 ( k ) 个结果时。

3. 召回率（Recall）

定义：召回率衡量从所有相关文档中成功检索到的比例，反映检索系统的覆盖范围。

$$ \text{Recall} = \frac{\text{检索到的相关文档数}}{\text{知识库中相关文档总数}} $$

变体：Recall@k
仅考虑前 ( k ) 个检索结果中的相关文档比例： $$ \text{Recall@k} = \frac{\text{前k个结果中相关文档的数量}}{\text{知识库中相关文档总数}} $$

适用场景：适用于评估检索系统是否遗漏了大量相关文档。

4. F1 分数

定义：F1 分数是精确率和召回率的调和平均值，用于平衡检索质量和覆盖范围。

$$ \text{F1} = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} $$

适用场景：适用于需要同时考虑精确率和召回率的场景，避免单一指标过高而忽略整体性能。

5. 平均倒数排名（Mean Reciprocal Rank, MRR）

定义：MRR 衡量一组查询中，每个查询的第一个相关文档的排名倒数的平均值。

$$ \text{MRR} = \frac{1}{N} \sum_{i=1}^{N} \frac{1}{\text{rank}(i)} $$ 其中，( N ) 为查询总数，( \text{rank}(i) ) 为第 ( i ) 个查询中第一个相关文档的排名。

适用场景：适用于评估系统快速找到第一个相关文档的能力。

6. 平均精确率均值（Mean Average Precision, MAP）

定义：MAP 结合不同截断级别（( k ) 值）的精确率和召回率，综合评估排序质量。

$$ \text{AP} = \frac{1}{R(i)} \sum_{k=1}^{n} \text{Precision@k} \times \text{rel}(k) $$ 其中，( R(i) ) 为查询 ( i ) 的相关文档总数，( \text{rel}(k) ) 为第 ( k ) 个文档的相关性（0 或 1）。

$$ \text{MAP} = \frac{1}{N} \sum_{i=1}^{N} \text{AP}(i) $$ 其中，( N ) 为查询总数。

适用场景：适用于需要综合考虑排序质量的场景。

7. 归一化折损累积增益（Normalized Discounted Cumulative Gain, nDCG）

定义：nDCG 衡量检索结果的排序质量，为较早出现的相关文档分配更高的权重，适用于文档具有不同程度相关性的场景。

$$ \text{DCG} = \sum_{i=1}^{n} \frac{2^{\text{rel}(i)} - 1}{\log_2(i+1)} $$ 其中，( \text{rel}(i) ) 为第 ( i ) 个文档的相关性分数。

$$ \text{IDCG} = \text{完美排序情况下的 DCG} $$

$$ \text{nDCG} = \frac{\text{DCG}}{\text{IDCG}} $$

适用场景：适用于需要精细评估排序质量和文档相关性程度的场景

改进RAG

改进召回率低的RAG

问题定位: 向量搜索基于语义相似性进行搜索，虽然能够捕捉到语义相关的文档，但在某些场景下无法精确匹配具体的关键词或结构化信息，导致召回率（Recall）较低。

改进方案:

在索引阶段，使用大语言模型（LLM）对数据进行预处理，提取结构化信息并优化索引字段，从而提升检索的精确性和召回率。

  1. 数据预处理与清洗：
    • 使用LLM对自由文本进行清洗，去除拼写错误、语言变体等噪声，提升数据的一致性。
  2. 结构化信息提取：
    • 对于可以结构化的数据: 使用LLM从自由文本中提取关键信息，生成结构化的服务列表。
  3. 用户查询优化：
    • 在用户查询阶段，使用LLM对用户输入进行关键词提取和结构化编排。

GraphRAG

由微软开源的一个RAG框架, 运用到了一些知识图谱的信息

传统RAG因为检索的都是零散的片段, 对于全局信息的掌控不足, GraphRAG可以解决这部分问题
